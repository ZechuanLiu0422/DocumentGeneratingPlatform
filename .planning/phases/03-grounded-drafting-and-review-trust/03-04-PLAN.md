<interfaces>
Chosen candidate lifecycle contract:

```text
drafts.pending_change jsonb = authoritative ephemeral preview state
```

Minimum stored fields:

```text
candidate_id, user_id, action, target_type, target_section_ids, created_at, expires_at,
base_updated_at, before, after, changed_section_ids, diff_summary
```

Lifecycle rules:

```text
- Preview overwrites any prior pending_change for the same draft/user.
- Preview creates no version snapshot.
- Accept requires matching user_id, unexpired expires_at, and unchanged base_updated_at.
- Reject requires matching user_id and clears pending_change.
- Accept clears pending_change and writes exactly one accepted version snapshot.
- Reject writes no version snapshot.
- Expired or stale-base candidates return a normalized stale-candidate error.
```
